# Midzoe Backend API — Implementation Guide

> **Base URL:** `https://midzobackend.vercel.app/api`
> **Auth:** Bearer token (JWT) via `Authorization: Bearer <token>` header
> **Content-Type:** `application/json`

---

## Registration Flow (2 steps)

```
1. POST /auth/register     → creates account, sends verification email, returns { success: true }
2. POST /auth/verify-email → validates code, returns { success: true, token, user }
```

After step 2 the frontend stores the token and logs the user in automatically.

---

## Auth Routes

### POST `/auth/register`

Create a new user account and send a verification email with a 6-digit code.

> **Note:** The frontend auto-generates the `username` from the email address (part before `@`, special characters replaced with `_`). No username input is shown to the user.

**Request Body**

| Field        | Type   | Required | Constraints                           |
|-------------|--------|----------|---------------------------------------|
| `username`  | string | ✅        | Auto-generated by frontend — unique   |
| `email`     | string | ✅        | Valid email format, unique            |
| `password`  | string | ✅        | Min 6 characters                      |
| `first_name`| string | ❌        | Min 2 chars                           |
| `last_name` | string | ❌        | Min 2 chars                           |
| `phone`     | string | ❌        | E.g. +33612345678                     |

**Example Request**
```json
{
  "username": "john_doe",
  "email": "john@example.com",
  "password": "secret123",
  "first_name": "John",
  "last_name": "Doe",
  "phone": "+33612345678"
}
```

**Success Response — 201 Created**
```json
{
  "success": true
}
```
> ⚠️ **No token is returned here.** The account is created but the user is not yet authenticated. A 6-digit code is sent to the provided email.

**Error Responses**

| HTTP Status | `error` value                 | Trigger                          |
|-------------|-------------------------------|----------------------------------|
| 400         | `"email already exists"`      | Email already registered         |
| 400         | `"username already exists"`   | Username collision (auto-gen)    |
| 400         | `"email is required"`         | Missing required field           |
| 400         | `"password is required"`      | Missing required field           |
| 400         | `"password too short"`        | Password < 6 chars               |
| 400         | `"invalid email format"`      | Malformed email                  |
| 500         | `"internal server error"`     | Unexpected server error          |

**Error Response Format**
```json
{
  "success": false,
  "error": "email already exists"
}
```

> **Frontend behavior:**
> - Error contains `"email"` → "An account with this email already exists."
> - Any other error → generic registration error message.

### POST `/auth/verify-email`

Validate the 6-digit code sent to the user's email. On success, returns a token and logs the user in.

**Request Body**

| Field   | Type   | Required | Constraints           |
|--------|--------|----------|-----------------------|
| `email` | string | ✅        | The registered email  |
| `code`  | string | ✅        | 6-digit numeric code  |

**Example Request**
```json
{
  "email": "john@example.com",
  "code": "482931"
}
```

**Success Response — 200 OK**
```json
{
  "success": true,
  "token": "<jwt_token>",
  "user": {
    "id": 42,
    "username": "john_doe",
    "email": "john@example.com",
    "first_name": "John",
    "last_name": "Doe",
    "phone": "+33612345678"
  }
}
```

**Error Responses**

| HTTP Status | `error` value             | Trigger                            |
|-------------|---------------------------|------------------------------------|
| 400         | `"invalid code"`          | Code does not match                |
| 400         | `"code expired"`          | Code is older than expiry window   |
| 404         | `"user not found"`        | Email not registered               |
| 400         | `"already verified"`      | Email already verified             |

### POST `/auth/resend-verification`

Resend a new 6-digit verification code to the user's email.

**Request Body**

| Field   | Type   | Required |
|--------|--------|----------|
| `email` | string | ✅        |

**Example Request**
```json
{
  "email": "john@example.com"
}
```

**Success Response — 200 OK**
```json
{
  "success": true
}
```

**Error Responses**

| HTTP Status | `error` value         | Trigger                              |
|-------------|-----------------------|--------------------------------------|
| 404         | `"user not found"`    | Email not registered                 |
| 400         | `"already verified"`  | Email already verified               |
| 429         | `"too many requests"` | Rate limit — suggest 60s cooldown    |

### POST `/auth/login`

Authenticate an existing verified user.

**Request Body**

| Field      | Type   | Required |
|-----------|--------|----------|
| `username` | string | ✅        |
| `password` | string | ✅        |

**Example Request**
```json
{
  "username": "john_doe",
  "password": "secret123"
}
```

**Success Response — 200 OK**
```json
{
  "success": true,
  "token": "<jwt_token>",
  "user": {
    "id": 42,
    "username": "john_doe",
    "email": "john@example.com",
    "first_name": "John",
    "last_name": "Doe",
    "phone": "+33612345678"
  }
}
```

**Error Responses**

| HTTP Status | `error` value              | Trigger                                  |
|-------------|----------------------------|------------------------------------------|
| 401         | `"invalid credentials"`    | Wrong username or password               |
| 403         | `"email not verified"`     | Account exists but email not verified    |
| 400         | `"username is required"`   | Missing field                            |
| 400         | `"password is required"`   | Missing field                            |

## Verification Code — Implementation Notes

- Generate a cryptographically random **6-digit numeric code** (e.g. `Math.floor(100000 + Math.random() * 900000)`)
- Store it in the database alongside the user record with a **creation timestamp**
- Code expiry: **15 minutes** recommended
- Hash the code before storing (bcrypt or SHA-256) to avoid leaking it from the DB
- Send the code via **transactional email** (SendGrid, Resend, Nodemailer, etc.)
- Email subject suggestion: *"Your Midzoe verification code: 482931"*
- After successful verification, mark the user as `email_verified = true` and delete/invalidate the code

---

## Token

- JWT signed with a secret key (`process.env.JWT_SECRET`)
- The frontend stores the token in `localStorage` under the key `midzo_token`
- The frontend stores the user object in `localStorage` under the key `midzo_user`
- Recommended expiry: **7 days**
- Include in every protected request: `Authorization: Bearer <token>`
